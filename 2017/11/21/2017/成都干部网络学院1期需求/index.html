成都干部网络学院1期需求

发表于 2017-11-21 | 分类于 需求
成都干部网络学院1期需求整理稿。

1期目标（2018年3月1日）：
实现网络培训的基础功能（含PC端和移动端），建成培训信息管理、学习管理、考试管理、系统管理、便捷移动管理、资源智能校审子系统，搭建学习资源公共服务平台、23个分平台。

2期目标（2018年10月1日）：
拓展和完善平台功能，完成干部学习大数据、人脸识别、学习文库、自适应学习、社群化学习、互动式直播、学习积分系统、人工智能客服等专属模块。

PC管理端

所有列表展现的数据，都可全选或取消全选，可显示总数据条目，可导出为excel。

分平台管理

每个分平台的管理员可以检索到所有的课程。

基本信息

    • 进行平台、单位的管理。
    • 设置人员及角色
    • 分平台的增删改查
    • 设置平台上下级关系，自定向下分别是：总平台、1级平台、2级平台、单位。
    • 管理各个平台下的学员
    • 新增子平台时，可以从已有学员数据中选择平台负责人，如果没有，则可以在此处直接新增平台负责人。

学员配置

此处实现每个被选择平台的学员配置。

    • 搜索列表功能
        • 平台下的学员明细列表需要能够快速搜索，可以导出搜索结果，也可以将搜索结果中的部分学员，移出平台。
        • 支持搜索记忆，展示本地的最近搜索历史。

课程配置

此处实现课程分类，审批通过且上线的某时间段的课程配置。

    • 选择课程分类，可以设置课程分类多选，也可以设定配置符合课程上线的起始时间的课程。

配置学员手机端入口

    • 后台可以修改移动学员端系统引导页的图片，以及配置跳转的内容（公告、资讯、课程）。

信息管理

公告资讯

单条资讯/公告包含：标题、内容、所属栏目、资讯标签、设置置顶、上下线、预览(PC、手机）、发布（指定发布人群、定时发送）、阅读量、

    • 公告/资讯管理
      • 筛选出列表，筛选条件：栏目、未发布、已发布。
      • 支持关键字模糊搜索。
      • 默认按照时间倒序排列。
      • 手动按照时间段（日、周、月、年）的阅读热度排序。时间段可自定义。
      • 一键发布、上线、下线筛选结果列表中的被选项。
      • 支持资讯多级栏目的创建和管理。

学员培训档案

培训档案包含：班级名称、开班时间、培训形式（线上、线下、综合）、承接单位、获得学时、课程清单。
课程清单包含：课程名称、主讲人、课程时长、学时。

学员在手机端或者PC学员端建立线下的课程，由单位管理员根据课表（课表对系统是未知的）来审核，系统根据学院填写的标题和课程关键字自动识别。课程分类，没有识别出来的分类自动归类到其他分类。管理员在审核的时候，可以修改没门课程对应的分类、承接单位。学员在手机端上传照片、审核通过后认可学时，学时才会被计入档案。

    • 建立学员培训档案
      单位管理员可以通过模板来导入培训档案。
    • 查看学员培训档案
      本模块数据统计 / 学员配合档案分析 模块是一致的，此处留有入口，点击查看学员培训档案，跳转过去。

调查问卷

发布调查问卷（筛选问卷），指定发布范围和对象（基于平台、班级、单位、学员、级别、年龄等）、查看问卷结果。
调查问卷支持两种模式：普通问卷模式、投票模式。在投票模式下，问卷只能包含一个单选或多选的题目，有多个选项。

    • 问卷列表
      筛选条件：标签、时间段
    • 问卷详情
      1、问卷结果：各选项占比
      2、导出问卷及原始数据
    • 新增问卷
      1、内容格式（单选、多选）
      2、发布范围（平台、班级）
      3、一个IP只允许参与一次
      4、是否允许浏览者查看问卷结果

学习管理

课程管理

课程属性：
名称、简介、主讲人、主讲人简介、课程缩略图地址、课程图片地址、时长、学时、点赞数、是否上线、课程标签ID、课程类型、课程标签、选课日期、上传时间、知识标签、课程来源

    • 课程管理
      • 筛选条件：课程分类、关键词、上线时间区间、年份。
      • 搜索条件：名称、主讲人。
      • 查询结果可以导出为Excel。包含审核信息。
      • 批量管理（上下线、删除）。
      • 试卷列表
        生成试卷、批量管理试卷、试卷预览
      • 课程预览
      • 管理课程评价
        上线、下线、删除
      • 课程题库管理
        1、本课程题库维护：管理和导入单选、多选、是非、填空、问答题型。可以纠错单个试题。
        2、试题的筛选，编辑、修改、上下线、删除、导出。
    • 课程专题
      • 可以将专题推送给某个或多个平台、班级，（已结束的班次不出现在推送列表中）。
      • 选择课程，组建专题
      • 自定义专题名称、专题图片
      • 上线、下线专题
      • 预置本地特色、精品课程专题集。这两个专题不可下线、不可删除。
      • 精品专题集仅总平台管理员可见，只有总平台管理员才可以将专题推送deodorant各个分平台/班级。
      • 课程评分可进行管理（手动调整基准评分等）。
    • 课程分类管理
      建立和维护多级分类。根据课程标题和简要介绍中根据关键字自动匹配或识别课程分类。
      分类和对应的关键字参考如下：
      https://shimo.im/doc/Tz4EWSkT5TMEYL8c?r=XODPGG/「【2016.3.25】课件分类与关键词」
    • 课程审核
        • 待我审核有消息通知。消息通知首页可见。
        • 通过和拒绝，需要填写审课意见。
        • 审课流程，管理员导入课程后，分配课程给教研部主任，教研部主任再次分配给指定的多个审核老师，审课老师对课程进行评分，并且提交课程可用状态，审课老师提交完毕后，教研部主任进行复核，填写相关意见。最后由平台管理员进行上线。流程中相关环节的人员都可以收到系统通知消息。
        • 管理员可以一键分配（所有审课员可以看到本课程）课程到教研部主任。教研部主任可以一键分配课程给审核老师。
        • 查看审课状态和导出评分表及评分汇总表。两个模板：审课评分表和评分汇总表。审核评分表模板和评分汇总表模板请参考链接：http://pan.baidu.com/s/1slkImzb 密码：7fc1

    • 课程推荐
      • 推荐规则设计如下：
        查询学员所在的平台的课程的关键词匹配情况。如果有匹配的，但是学员没有学习过的，就推荐给学员。学员收到推荐消息后，可以在被推荐的课程上点击不感兴趣。对应的具体课程就不会下次被推荐。推荐周期根据平台的课程库每周一次。每次推荐10条（包含了手动和自动推荐课程，手动优先），按照课程上传的时间由近及远排列。推荐课程先选择评分高、点赞数、播放量高的课程。如下系统推荐课程的3点，推荐的比例可以手动设置设置。

        2、根据知识储备图，推荐缺失的一部分给学员。

班级管理

  • 超管可以管理所有班级。
  • 平台管理员可对具体课程进行一键听课设置。
  • 选入和修改学员，对学员进行分组。
  • 班级管理员可以管理名下班级。
  • 单位管理员可以查看本单位人员所在班级的信息（只能看到本单位人员的数据）。
  • 搜索筛选条件：平台、开班时间段、名称搜索、未开始、进行中、已结束。
  • 操作：
    • 班级增删改
      • 编辑班级时，支持选入和修改学员。
      • 班级支持设置考试方式：
        • 支持选择并安排多个考试
        • 支持选择一个调查问卷。
      • 设置是否由班主任来批阅和推荐学员的学习心得。
    • 查看本班级下学员学习整体记录明细。
      • 查看每门课程的学员总体学习进度（已完成学员数/未完成学员数）。
      • 查看每个学员的学习进度（已完成/未完成课程数量）。

  • 考试策略

    • 设置班级下所有必修课的考试策略。包含：考试时间、出题策略（是否打乱题序、是否打乱答案顺序、题目类型的分数占比，试卷可以预览）、考试次数（默认无限次）。同时需要记录学员在每门课程上的考试次数。
    • 每门必修课都进行一次考试。记录每个学员的考试次数。
    • 支持的操作：批量强行提交、发送警告、删除考试记录、增加考生、一键考试通过。
    • 查看考试明细表
      1、属性：考试时间、得分、考试范围（一个或者多个班级）、考试时间、开卷、闭卷
      2、操作：查看答卷、强行提交、发送警告、删除答卷、推送考试通知（短信、消息）报考人员列表、延长考试时间。
      • 可以查看到每门课程的成绩分布饼图。

  • 调查问卷

    • 支持选择、新增一个调查问卷，发布到本班级。调查问卷详情请参考：
  • 学习心得
    • 班主任来批阅和推荐学员的学习心得。学习心得支持推送到管理员可见的平台上。
  • 发布教学公告
  • 立即/定时发送学习提醒
  • 发布必修课、选修课学习时间提醒，可以手动发送和定时发送。支持短信和消息发送。
  • 班主任可对某学员设置一键听课
  • 配置必修课和选修课，导出班级的课程配置清单。课程配置页面支持筛选，筛选条件：分类、标签、模糊检索（班级名称）
      • 必修课设为先看完之后，才能进行考试；或者不用看，就能进行直接考试两种。
  • 可以查看班级任一学员的学习进度
  • 可发送调查问卷。
    • 学员对教师打分
    • 教师对学员打分（只针对线下）
  • 出题策略管理
    • 是否打乱出题顺序（每个学员看到的不同）
    • 设置题目类型，数目及分值
      设置出题规则：单选、多选、是非、填空、简答各占多少道，占多少分。
    • 设置考试次数、及格线、开卷、闭卷
    • 生成试卷，预览题目
    • 试卷管理
      1、设定名称、选择出题规则、是否打乱题序
      2、试卷预览
      3、上线、下线
  • 证书管理
    1、电子证书打印（支持批量套打功能）
    2、支持水印、唯一证书编号、显示班级名称、时间段、成绩等。
    3、支持在线查询

数据统计

培训档案统计

  • [ ] 可筛选已完成/未完成的班级中，每个学员在班级的进度，每门课程的进度。
    只统计完成的学习班数据。每个临时筛选出来的数据都可以导出。

  • 学员培训档案统计
    先筛选出一位学员，然后展示统计表格。
    表格标题：班级名称、时间段、培训形式（线上、线下、自定义）、承接单位（线下有承接单位，线上班没有）、获得学时、操作（查看课程明细）、考核结果。
    柱状图表横轴统计数据为：按班级、按年度、按平台、按单位、按线上线下；纵轴统计数据为：获得学时、累计学习时长、登录次数。

  • 班级培训档案统计
    表格标题：班级、人数、通过率、必修课门数、选修课门数、考核要求学时。

  • [ ] 不同学习班的优秀、良好、合格、不合格的数据对比柱状图。

  • 单位培训档案统计
    统计表格标题：年份、班级名称、参加人数、通过率（）。
    通过率是针对班级来说的。有多少人通过，多少人未通过。

学员登录后要首先选择进入哪个班级。
2、此页留有管理员的入口。
3、展示各平台的历史浏览量、当前在线人数。
4、统计学时数最高的前10名单位。
5、精品专题、本地特色的课程学员登录后就可以自学。如果某学员自学过的课程，后续又被其所在的班级采纳，则已经学习过的记录会被转换为班级要求的学时，学员不用再次学习。

门户页包括3个资讯入口：最新资讯、权威新闻、干教动态。1个功能入口：学员天地。
如上4个入口都是在管理端进行设置。每个分平台都可以进行对应模块的内容设置。

学员在平台门户页可以进行课程选学，以后如果被分配到了班级的必修课和选修课是，不用再次学习。

学员天地

功能首页

1、用户登录后，首先选择班级。每个班级的推荐专题、特色课程等都是独立的。
2、显示平台名称信息。
3、统计本班级学时数最高的前10名学员。

消息提醒

包含：公告消息、考试消息、审课消息、推荐课程消息、其他消息.

    • 学员收到课程推荐消息后，可以在被推荐的课程上点击不感兴趣。

移动管理端

学员管理

1、筛选学员列表：根据单位、平台、班级筛选出学员列表。
2、点击上一步的筛选结果，编辑学员。也可以查看具体学员的学习档案。
3、PC端可以批量导入学员，移动端不可以。
4、为学员分配班级，可批量操作；查看学员用户名、密码、单位信息。

  • 学习档案列表（历年的学习历史记录）
    1、根据平台、开班时间和结束时间区间筛选到对应的学习班列表，列表中展示对应班级的基本信息，以及获取的总学时。
    2、点击上一步的筛选结果，选中其中一个班，点击后打开班级下的课程列表，列出每门课程的基本信息，培训方式，以及获取的总学时。
    3、可以查看每个学员在每个班级的进度，每门课程的进度。
